## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Screenshots or proof-of-concept (if helpful)
- The browser and device used

### How to report:
- Open a **private issue** (if available), or
- Contact the site owner directly

Do **not** publicly disclose vulnerabilities before they are reviewed and fixed.

---

## Scope

This security policy applies to:
- Client-side JavaScript
- HTML/CSS
- Local storage usage
- Fullscreen iframe behavior

This website:
- Does **not** collect personal data
- Does **not** use accounts or servers
- Does **not** process payments
- Does **not** store passwords remotely

All authentication is handled locally in the browser.

---

## Known Limitations

- Some third-party websites may block iframe embedding using browser security headers.
- The site cannot control or modify third-party content behavior.
- Browser extensions or modified browsers may affect security behavior.

---

## Responsible Disclosure

We appreciate responsible disclosure and will make reasonable efforts to address valid security concerns in a timely manner.

Thank you for helping keep this project safe.
